# 통합 흐름 개요

이 문서는 기업 등록과 지원이력 대량 등록의 **사용자 흐름**이 동일한지 확인하고,
차이점은 데이터 처리 로직만 다름을 명시합니다.

## 공통 사용자 흐름 (6단계)
1. **파일 다운로드** - "Excel 다운로드" 버튼 클릭 → 파일 선택 → .xlsx 파일 로드
2. **파싱 시작**  
   - FileReader로 배열 버퍼 로드 → `parseBulkExcel` 함수로 객체 배열 변환
   - 반환 타입: `BulkPreviewEntry[]`
3. **미리보기 표시**  
   - `<BulkPreviewTable items={bulkPreview} />`  
   - 컬럼은 기업등록: `기업명`, `사업자번호`, `등록여부`
   - 지원사업: `기업명`(매칭된 것), `사업자번호`, `매칭기업`, `지원금액`, `지원과제명`, `선정금액`, `지원금액`, `비고`
4. **업로드 수행**  
   - "업로드 진행" 버튼 클릭 → `handleUpload(entries)` 호출
   - API: `POST /api/bulk-upload` (공통)
5. **결과 확인**  
   - 성공/실패 알림 Toast 표시
   - 성공/실패 메시지는 텍스트만 다르며, Toast UI 자체는 동일

> **핵심**: UI/UX 단계는 완전히 동일합니다. 데이터 매핑 차이만 존재합니다.

## 검색 UI 흐름

### 단일검색
1. **검색어 입력** - 기업명 또는 사업자번호 입력 (라벨에 `*` 필수 표시)
2. **검색 실행** - 300ms 디바운스 후 DB 전체 대상 검색
3. **결과 표시** - SearchResultModal에서 내부DB 매칭 정보 표시

### 대량검색 (엑셀 업로드)
1. **필수 입력값 작성** - 지원사업명(*), 요청기관(*), 문서번호(*), 접수일(*), 요청일(*)
2. **파일 업로드** - 엑셀 파일 드롭존 업로드
3. **파싱 및 미리보기** - 고정 컬럼 인덱스 기반 파싱 → 테이블 표시
4. **업로드 수행** - `handleUpload` 호출, 지원사업명은 DB `excel_uploads.program_name`에 저장
5. **결과 확인** - Toast 알림

## 페이지 라우트 구조

| 경로 | 헤더 타이틀 | 설명 |
|------|-------------|------|
| `/search` | 통합검색 | 메인 대시보드 (단일/대량 검색) |
| `/database` | 등록기업 | 기업 DB 관리 (신규기업등록/지원이력등록 버튼) |
| `/request` | 조회요청기업 | 엑셀을 통해서 조회 요청된 기업의 목록 |
| `/file` | 엑셀관리 | 업로드된 엑셀 파일 관리 |
| `/verify` | 검증결과 | 검증 결과 목록 |
| `/history` | 검색 기록 | 검색 이력 관리 |
