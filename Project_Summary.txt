# SearchCompany 프로젝트 전체 작업 산출물 및 요약 보고서

## 1. 프로젝트 개요
- **프로젝트명**: SearchCompany
- **목적**: 기업 정보 관리, 검색, 매칭 및 히스토리 관리 등을 제공하는 웹 대시보드
- **기술 스택**: 
  - **Frontend**: Next.js (App Router), React, TypeScript, Vanilla CSS
  - **Backend/Database**: Supabase (PostgreSQL)
  - **Deployment**: GitHub Pages, Vercel

## 2. 주요 구현 기능
- **대시보드 (Dashboard)**: 기업 검색, 최신 업데이트 내역, 빠른 작업 바로가기 제공
- **데이터베이스 관리 (Database)**: 
  - 새로운 기업 정보 등록 및 기존 기업 정보 수정
  - Excel 기반 대량 데이터 업로드 및 다운로드 지원 (템플릿 제공)
- **히스토리 관리 (History)**: 기업 검색 및 매칭 관련 과거 기록 조회 기능
- **디자인 및 UI/UX**: 제공된 `Design.md` 및 `_sample_design`을 참고하여 고품질의 Glassmorphism 및 모던 기업형 UI 구현

## 3. 백엔드 및 데이터베이스 연동 (Supabase)
- **스키마 구축**: 기업 테이블(`companies`), 사용자 테이블(`users`), 로그/히스토리 테이블(`history`) 등 구성 (`supabase_schema.sql` 기반)
- **보안 및 환경변수**: `.env.local`을 통해 클라이언트 키(`NEXT_PUBLIC_SUPABASE_ANON_KEY`)와 URL을 연동

## 4. 배포 내역 및 상태 (Deployment)
### 4.1 GitHub Repository
- **저장소 위치**: https://github.com/kelvinks/SearchCompany
- **상태**: Public 저장소로 설정 완료
- **버전 관리**: `.gitignore`가 적용된 상태로 전체 소스코드 `main` 브랜치에 안전하게 병합

### 4.2 GitHub Pages 배포
- **URL**: https://kelvinks.github.io/SearchCompany/
- **방식**: GitHub Actions를 통한 CI/CD 파이프라인 구축 (`.github/workflows/deploy.yml`)
- **설정**: `next.config.ts` 파일 내 `output: "export"` 및 동적 `basePath` 설정을 통해 정적 사이트로 빌드 및 배포

### 4.3 Vercel 배포 (선택적 병행)
- **URL**: https://frontend-xi-ten-ray29dy8qw.vercel.app/
- **방식**: Next.js 기본 호환성을 활용하여 Vercel CLI로 프로덕션 빌드 배포

## 5. 추후 유지보수 가이드
1. **코드 수정 후 배포**: `main` 브랜치에 커밋 후 `git push`를 진행하면 GitHub Actions가 자동으로 빌드를 시작하여 GitHub Pages에 반영됩니다.
2. **환경변수 관리**: Supabase 등 외부 API 키가 변경될 경우 GitHub Repository의 `Settings > Secrets and variables > Actions`에서 `NEXT_PUBLIC_SUPABASE_URL` 및 `NEXT_PUBLIC_SUPABASE_ANON_KEY`를 업데이트해야 합니다.
3. **로컬 실행**:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
   이후 `http://localhost:3000`에서 로컬 개발 버전을 확인할 수 있습니다.
